# 要件定義

## 必須機能

最初のバージョンでは、次の機能を実装します。

- 企業名または銘柄名の入力
- 投資を考えた理由の入力
- 参考にした情報源の選択
- 基本的な確認チェックリスト
- リスク診断結果の表示
- 追加で確認すべき項目の表示

## 情報源の選択肢

利用者は、次の情報源を選択できるようにします。

- SNS
- YouTube
- ニュース記事
- 友人・知人
- 企業IR
- 決算資料
- 株価チャート
- 配当利回り
- アナリストレポート
- その他

## 確認項目

最初のバージョンでは、次のようなシンプルな質問を用意します。

- その企業の主な事業を理解しているか
- 直近の業績を確認したか
- リスク要因を確認したか
- 競合企業と比較したか
- なぜ株価が動いたのかを確認したか
- ひとつの情報源だけに頼っていないか

## リスク分類

アプリでは、次のようなリスクを表示します。

- 情報源の偏りリスク
- 話題性依存リスク
- 感情的判断リスク
- 事業理解不足リスク
- 業績確認不足リスク
- 単一情報源依存リスク

## 出力内容

診断結果画面では、次の内容を表示します。

- 利用者の判断傾向の短い要約
- リスクラベル
- 分かりやすい説明
- 追加で確認すべき項目
- 金融アドバイスではないことの注意書き

## 追加機能

最初の完成には必須ではありませんが、余裕があれば追加したい機能です。

- 診断結果の保存
- 診断結果のMarkdown出力
- 簡単なグラフ表示
- 初心者向け用語集
- サンプル診断

## 将来機能

MVP完成後に検討する機能です。

- 株価API連携
- 企業IRリンクの提案
- ニュース検索連携
- AIによる入力内容の要約
- 複数企業の比較

## 非機能要件

- 初心者にも分かりやすいこと
- 専門用語や金融用語をできるだけ避けること
- MVPでは機密性の高い個人情報を保存しないこと
- アプリの限界を明確に説明すること
