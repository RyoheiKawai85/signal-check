# 企画書

## プロジェクト名

SignalCheck

## 概要

SignalCheckは、株式投資に関心を持ち始めた初心者が、投資判断に使っている情報を整理し、自分の判断が偏った情報や感情に依存していないかを確認するためのPython製Webアプリです。

目的は、利用者に「何を買うべきか」を教えることではありません。利用者が、自分の判断の前提を見直し、追加で確認すべきことに気づけるようにすることです。

## 想定ユーザー

- 投資に興味を持ち始めた学生
- NISAや株式投資を学び始めた若年層
- SNS、YouTube、ニュース、友人の話などを参考にしながら投資を考えている初心者
- 投資判断の前に、自分の考えを整理したい人

## 開発動機

投資に関する情報は、以前よりも簡単に手に入るようになっています。しかし、情報が多いことは、必ずしも良い判断につながるとは限りません。

初心者は、SNSで話題になっている銘柄を見たり、友人から聞いたり、株価が上がっていることを見たりして、その企業が魅力的に見えることがあります。しかし、その時点では事業内容、業績、リスク要因、株価が動いた理由などを十分に確認できていない場合があります。

このプロジェクトは、初心者が投資判断の前に一度立ち止まり、自分が何を根拠に考えているのかを整理する必要がある、という問題意識から始めます。

## 最初の完成ライン

最初のバージョンでは、次の状態を目指します。

- 企業名または銘柄名を入力できる
- 参考にした情報源を選択できる
- 投資を考えた理由を入力できる
- 情報源の偏りを簡易診断できる
- リスク分類を表示できる
- 追加で確認すべき項目を提示できる

## 開発方針

- 最初のバージョンは小さく作る
- 金融アドバイスに見えない設計にする
- 初心者にも理解しやすい説明にする
- 課題定義とリスク診断モデルをGitHub上に明文化する
- PythonとStreamlitを使い、段階的に開発する

## 開発フェーズ

### Phase 1: 企画立案

プロジェクト名、対象ユーザー、解決したい課題、開発動機、最初の完成ラインを定義します。

### Phase 2: 機能分解

必須機能、追加機能、将来機能に分け、最初に作る範囲を絞ります。

### Phase 3: 技術構成

PythonとStreamlitを使います。最初から本格的なWeb開発にせず、Python中心で画面つきアプリを作れる構成にします。

### Phase 4: GitHub準備

READMEとdocsを作成し、企画や設計の過程がGitHub上で見えるようにします。

### Phase 5: MVP開発

入力フォーム、リスク診断ロジック、診断結果画面を作ります。

### Phase 6: レビューと改善

アプリが初心者に分かりやすいか、投資助言に見えないか、限界や注意点を明確に説明できているかを確認します。
